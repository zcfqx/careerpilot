### 1.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 前后端分离 | 前端小程序通过HTTP API与后端通信，完全解耦 |
| 分层架构 | 后端采用路由→控制器→服务→模型四层架构 |
| 单一职责 | 每个模块/文件只负责一个功能领域 |
| 配置外置 | 敏感配置通过.env文件管理，不硬编码 |
| 渐进增强 | MVP先跑通核心流程，后续迭代逐步增强 |

### 2.4 网络请求封装设计

```javascript
// utils/api.js 核心设计
const BASE_URL = 'http://localhost:3000/api';

const request = (options) => {
  return new Promise((resolve, reject) => {
    wx.request({
      url: BASE_URL + options.url,
      method: options.method || 'GET',
      data: options.data || {},
      header: {
        'Content-Type': 'application/json',
        'Authorization': wx.getStorageSync('token') || ''
      },
      success: (res) => {
        if (res.statusCode === 200) {
          resolve(res.data);
        } else {
          reject(res);
        }
      },
      fail: (err) => {
        wx.showToast({ title: '网络异常', icon: 'none' });
        reject(err);
      }
    });
  });
};
```

**各层职责：**

| 层级 | 职责 | 示例 |
|------|------|------|
| Route | 定义URL路径与HTTP方法，将请求转发到对应控制器 | `router.post('/assessment', controller.create)` |
| Controller | 接收请求、参数校验、调用服务、格式化响应 | 校验必填字段、捕获异常、返回JSON |
| Service | 核心业务逻辑、调用大模型、组装数据 | 构建Prompt、调用LLM、解析结果 |
| Model | 数据库操作（CRUD） | 查询岗位、保存评估记录 |

### 3.5 中间件设计

```javascript
// JWT鉴权中间件
const auth = (req, res, next) => {
  const token = req.headers.authorization?.replace('Bearer ', '');
  if (!token) {
    return res.status(401).json({ code: 401, message: '未登录' });
  }
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.userId = decoded.userId;
    next();
  } catch (err) {
    return res.status(401).json({ code: 401, message: '登录已过期' });
  }
};

// 统一错误处理中间件
const errorHandler = (err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({
    code: 500,
    message: '服务器内部错误',
    data: null
  });
};
```

### 5.3 大模型调用封装

```javascript
// utils/llm.js 核心设计
const axios = require('axios');

class LLMService {
  constructor() {
    this.provider = process.env.LLM_PROVIDER || 'qwen';
  }

  async chat(systemPrompt, userPrompt) {
    if (this.provider === 'qwen') {
      return this.chatWithQwen(systemPrompt, userPrompt);
    } else if (this.provider === 'glm') {
      return this.chatWithGLM(systemPrompt, userPrompt);
    } else if (this.provider === 'deepseek') {
      return this.chatWithDeepSeek(systemPrompt, userPrompt);
    }
    return this.chatWithQwen(systemPrompt, userPrompt);
  }

  async chatWithQwen(systemPrompt, userPrompt) {
    const response = await axios.post(
      'https://dashscope.aliyuncs.com/api/v1/services/aigc/text-generation/generation',
      {
        model: 'qwen-plus',
        input: {
          messages: [
            { role: 'system', content: systemPrompt },
            { role: 'user', content: userPrompt }
          ]
        },
        parameters: {
          result_format: 'json_object',
          temperature: 0.7
        }
      },
      {
        headers: {
          'Authorization': `Bearer ${process.env.QWEN_API_KEY}`,
          'Content-Type': 'application/json'
        }
      }
    );
    const text = response.data.output.text;
    return typeof text === 'string' ? JSON.parse(text) : text;
  }

  async chatWithDeepSeek(systemPrompt, userPrompt) {
    const response = await axios.post(
      'https://api.deepseek.com/v1/chat/completions',
      {
        model: 'deepseek-chat',
        messages: [
          { role: 'system', content: systemPrompt },
          { role: 'user', content: userPrompt }
        ],
        temperature: 0.7,
        response_format: { type: 'json_object' }
      },
      {
        headers: {
          'Authorization': `Bearer ${process.env.DEEPSEEK_API_KEY}`,
          'Content-Type': 'application/json'
        }
      }
    );
    const text = response.data.choices[0].message.content;
    return typeof text === 'string' ? JSON.parse(text) : text;
  }
}
```

   ```bash
   # 安装Node.js

   # 安装Python
   sudo apt install python3 python3-pip
   
   # 安装Nginx
   sudo apt install nginx
   
   # 安装PM2
   sudo npm install -g pm2
   ```
5. **上传代码**：通过Git拉取或SCP上传
6. **配置Nginx反向代理**
7. **微信小程序后台配置服务器域名**

## 7. 安全架构

### 7.1 认证流程

```
小程序端                        后端                         微信服务器
   │                            │                              │
   │  wx.login()                │                              │
   │ ─────────────────────→     │                              │
   │  返回code                  │                              │
   │ ←─────────────────────     │                              │
   │                            │                              │
   │  发送code到后端             │                              │
   │ ─────────────────────→     │  用code换取openid/session_key │
   │                            │ ──────────────────────────→   │
   │                            │  返回openid/session_key       │
   │                            │ ←──────────────────────────   │
   │                            │                              │
   │                            │  生成JWT token               │
   │  返回token + 用户信息       │                              │
   │ ←─────────────────────     │                              │
   │                            │                              │
   │  后续请求携带token          │                              │
   │ ─────────────────────→     │  验证token                   │
   │                            │                              │
```

### 7.2 安全措施清单

| 措施 | 说明 |
|------|------|
| HTTPS传输 | 生产环境所有通信使用HTTPS |
| JWT鉴权 | 接口请求需携带有效token |
| 参数校验 | 后端对所有入参进行类型和范围校验 |
| SQL参数化 | 使用sql.js的参数化查询，防止SQL注入 |
| 速率限制 | 限制单用户API调用频率 |
| 敏感信息加密 | 用户手机号等敏感信息加密存储 |
| .env保护 | 敏感配置不提交代码仓库 |
| CORS白名单 | 仅允许小程序域名跨域访问 |
