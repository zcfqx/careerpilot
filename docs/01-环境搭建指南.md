## 1. 整体技术栈概览

本项目采用**前后端分离**架构，技术选型如下：

| 模块 | 技术 | 说明 |
|------|------|------|
| 前端 | 微信小程序（原生框架） | 用户交互界面 |
| 后端 | Node.js + Express | 提供API接口服务 |
| 数据库 | SQLite | 轻量级嵌入式数据库，无需安装 |
| 大模型 | 通义千问 / 智谱GLM API | AI评估与推荐 |
| 爬虫 | Python + Scrapy | 采集招聘网站数据 |
| 部署 | 云服务器（阿里云/腾讯云） | 线上运行环境 |

**为什么这样选？**
- 微信小程序原生框架：学习成本最低，官方文档完善，适合小白
- Node.js + Express：JavaScript全栈统一语言，Express是最简洁的后端框架
- SQLite：轻量级嵌入式数据库，无需单独安装服务，适合开发和小型项目
- 通义千问/智谱GLM：国内大模型API，调用方便，有免费额度

## 2. 前端环境搭建（微信小程序）

### 2.1 注册微信小程序账号

⚠️ **这是需要手动操作的重要步骤！**

1. 打开浏览器，访问 [微信公众平台](https://mp.weixin.qq.com/)
2. 点击右上角「立即注册」
3. 选择账号类型为「小程序」
4. 填写邮箱（未绑定过微信公众号/小程序的邮箱）、密码
5. 邮箱激活后，选择主体类型为「个人」
6. 填写个人身份信息，完成注册
7. 注册成功后，登录小程序管理后台

### 2.2 获取小程序关键信息

⚠️ **手动操作：记录以下信息，后续开发必须使用**

1. 登录 [小程序管理后台](https://mp.weixin.qq.com/)
2. 左侧菜单点击「开发」→「开发管理」→「开发设置」
3. 记录 **AppID**（格式类似 `wx1234567890abcdef`）
4. 在「服务器域名」中配置后端API地址（开发阶段可先不配，在开发者工具中勾选「不校验合法域名」）

### 2.3 安装微信开发者工具

1. 访问 [微信开发者工具下载页面](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 选择 Windows 64位 稳定版（Stable）下载
3. 双击安装包，按提示完成安装（建议安装到默认路径）
4. 安装完成后，打开微信开发者工具
5. 使用微信扫码登录

### 2.4 打开已创建的小程序项目

**前端项目文件已经创建完成，不需要重新创建！**

1. 打开微信开发者工具，点击「导入项目」
2. 项目目录选择 `frontend` 文件夹
3. AppID：填入2.2节获取的AppID（或选择「测试号」）
4. 后端服务：选择「不使用云服务」
5. 点击「确定」导入项目

### 2.5 开发者工具常用设置

1. 点击右上角「详情」→「本地设置」
2. ✅ 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」（开发阶段必须勾选，否则无法调用本地后端）
3. ✅ 勾选「启用自定义处理命令」
4. 调试基础库选择最新稳定版

## 3. 后端环境搭建（Node.js + Express）

### 3.1 安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载 **LTS（长期支持版）**，版本号以偶数结尾（如 v20.x.x 或 v22.x.x）
3. 双击安装包，**全部保持默认选项**，一路点 Next
4. 安装完成后，验证安装：

```bash
# 打开命令提示符（按 Win+R，输入 cmd，回车）
node -v
# 应显示 v20.x.x 或更高版本

npm -v
# 应显示 10.x.x 或更高版本
```

### 3.2 配置 npm 镜像源

⚠️ **国内网络必须配置，否则安装依赖极慢或失败**

```bash
npm config set registry https://registry.npmmirror.com
```

验证配置：
```bash
npm config get registry
# 应显示 https://registry.npmmirror.com/
```

### 3.3 安装后端依赖

**后端项目文件已经创建完成，不需要重新创建！**

```bash
# 进入后端目录
cd backend

# 安装依赖（已在 package.json 中配置）
npm install
```

**依赖说明：**

| 包名 | 作用 |
|------|------|
| express | Web框架，处理HTTP请求 |
| sql.js | SQLite数据库驱动（纯JavaScript实现，无需编译） |
| cors | 处理跨域请求 |
| dotenv | 读取环境变量配置文件 |
| axios | HTTP请求库（调用大模型API用） |

### 3.4 验证后端环境

```bash
# 启动后端服务
cd backend
npm run dev
```

打开浏览器访问 `http://localhost:3000`，看到 `{"message":"AI韧性职业适配系统后端服务运行中"}` 即为成功。

## 4. 数据库环境搭建（SQLite）

### 4.1 SQLite 简介

SQLite 是一个轻量级的嵌入式数据库，具有以下特点：
- **无需安装**：数据库就是一个文件，不需要单独安装数据库服务
- **零配置**：不需要复杂的配置过程
- **跨平台**：支持所有主流操作系统
- **轻量级**：整个数据库引擎只有几百KB

### 4.2 数据库文件位置

项目数据库文件位于：`backend/data/ai_career.db`

**注意**：数据库文件会在后端首次启动时自动创建，无需手动创建。

### 4.3 数据库可视化管理工具（推荐）

推荐安装 **DB Browser for SQLite**（免费），方便查看和管理数据库：

- 下载地址：https://sqlitebrowser.org/dl/
- 安装后，打开软件，点击「打开数据库」
- 选择项目中的 `backend/data/ai_career.db` 文件
- 即可查看和编辑数据库中的数据

## 5. 大模型API接入准备

### 5.1 注册通义千问API（推荐）

⚠️ **手动操作**

1. 访问 [阿里云百炼平台](https://bailian.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 开通「模型服务灵积」
4. 进入「API-KEY管理」，创建新的 API Key
5. **记录保存 API Key**（只显示一次！）

### 5.2 或注册智谱GLM API

1. 访问 [智谱AI开放平台](https://open.bigmodel.cn/)
2. 注册账号
3. 进入「API Keys」页面，创建 API Key
4. **记录保存 API Key**

### 5.3 或注册DeepSeek API

1. 访问 [DeepSeek开放平台](https://platform.deepseek.com/)
2. 注册/登录账号
3. 进入「API Keys」页面，创建新的 API Key
4. **记录保存 API Key**（只显示一次！）

**DeepSeek 优势**：
- 性价比高，价格便宜
- 支持长上下文（128K）
- 中文理解能力强
- 兼容 OpenAI API 格式

### 5.4 配置环境变量

**已创建 `.env` 文件，位于 `backend/.env`**

打开 `backend/.env` 文件，填入你的 API Key：

```
# 大模型API配置（选择一个即可）
LLM_PROVIDER=qwen
QWEN_API_KEY=你的通义千问API_Key
# GLM_API_KEY=你的智谱API_Key
# DEEPSEEK_API_KEY=你的DeepSeekAPI_Key

# 数据库配置
DB_PATH=./data/ai_career.db

# 服务端口
PORT=3000

# 微信小程序配置
WX_APPID=你的小程序AppID
WX_SECRET=你的小程序Secret
```

**LLM_PROVIDER 可选值**：
- `qwen`：通义千问（默认）
- `glm`：智谱GLM
- `deepseek`：DeepSeek

⚠️ **`.env` 文件包含敏感信息，绝对不能上传到公开仓库！**

## 6. 爬虫环境搭建（Python）

### 6.1 安装 Python

```bash
# 应显示 Python 3.11.x 或更高

pip --version
```


## 8. 常见问题与排错

### Q1：npm install 报错 ECONNREFUSED
**原因**：npm镜像源未配置或网络问题
**解决**：
```bash
npm config set registry https://registry.npmmirror.com
npm cache clean --force
npm install
```

### Q2：微信开发者工具无法调用本地后端
**解决**：确认在「详情」→「本地设置」中勾选了「不校验合法域名」

### Q3：SQLite 数据库文件不存在
**原因**：后端首次启动时会自动创建数据库文件
**解决**：确保后端服务已启动，数据库文件会自动创建在 `backend/data/` 目录下

### Q4：Python 命令找不到
**原因**：安装时未勾选「Add Python to PATH」
**解决**：重新运行安装程序，勾选该选项；或手动添加 Python 安装路径到系统环境变量

### Q5：nodemon 不是内部或外部命令
**解决**：使用 `npx nodemon app.js` 代替 `nodemon app.js`
