## 1. 启动前检查清单

在启动项目之前，请确认以下内容已完成：

### 1.1 环境检查

```bash
# 检查 Node.js
node -v
# 应显示 v18.x 或更高版本

# 检查 npm
npm -v
# 应显示 9.x 或更高版本

# 检查 Python
python --version
# 应显示 Python 3.11.x 或更高
```

### 1.2 配置文件检查

确认 `backend/.env` 文件已正确配置：

```
LLM_PROVIDER=qwen
QWEN_API_KEY=你的通义千问API_Key
DB_PATH=./data/ai_career.db
PORT=3000
WX_APPID=你的小程序AppID
WX_SECRET=你的小程序Secret
```

### 1.3 数据库检查

SQLite 数据库文件会在后端首次启动时自动创建。确认 `backend/data/` 目录存在：

```bash
# 检查数据库目录
ls backend/data/

# 如果目录不存在，启动后端时会自动创建
cd backend
npm run dev
```

启动后应该能看到 `backend/data/ai_career.db` 文件已创建。

### 1.4 确认端口未被占用

```bash
# 检查 3000 端口是否被占用
netstat -an | findstr 3000
```

如果显示有其他程序占用 3000 端口，可以在 `.env` 中修改 `PORT=3001` 等其他端口。

## 2. 启动后端服务

### 2.1 开发环境启动

```bash
# 进入后端目录
cd backend

# 安装依赖（如果尚未安装）
npm install

# 启动开发服务器（支持热重载）
npm run dev
```

成功启动后，会看到类似输出：

```
[AI Career] 数据库初始化成功
[AI Career] 服务器已启动: http://localhost:3000
[AI Career] API文档: http://localhost:3000/api
```

### 2.2 验证后端服务

打开浏览器访问：`http://localhost:3000`

应该看到：

```json
{"code":200,"message":"AI韧性职业适配系统后端服务运行中","data":null}
```

### 2.3 测试API接口

访问：`http://localhost:3000/api`

应该看到：

```json
{"code":200,"message":"success","data":{"list":["auth","assessment","career","data"]}}
```

### 2.4 常见问题

**问题1：数据库初始化失败**
```
Error: Database not initialized
```
**解决**：检查 `backend/data/` 目录是否有写入权限

**问题2：端口被占用**
```
Error: listen EADDRINUSE :::3000
```
**解决**：修改 `.env` 中的 `PORT` 为其他端口，如 `3001`

## 3. 启动微信小程序

### 3.1 使用微信开发者工具

1. 打开 **微信开发者工具**
2. 点击「导入项目」
3. 项目目录选择 `frontend` 文件夹
4. AppID 填入你的小程序 AppID（或使用测试号）
5. 后端服务选择「不使用云服务」
6. 点击「确定」

### 3.2 开发者工具设置

首次打开项目，需要进行以下设置：

1. 点击右上角 **「详情」**
2. 选择 **「本地设置」**
3. ✅ 勾选 **「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」**
4. ✅ 勾选 **「启用自定义处理命令」**

### 3.3 编译运行

1. 在左侧模拟器中，应该能看到小程序的首页
2. 如果显示空白，检查控制台是否有报错
3. 点击「编译」按钮重新编译项目

### 4.2 AI评估

**位置**：小程序底部 tabBar 第2个「评估」

- 点击选择专业，或直接输入专业名称

- 工作风格：稳定/挑战/平衡

**步骤4：提交评估**
- 点击「提交评估」

## 5. 常见使用场景

### 场景1：首次使用，快速体验

1. 启动后端 `npm run dev`
2. 打开微信开发者工具，导入 frontend
3. 直接点击首页「开始AI评估」
4. 完成3步表单，提交评估
5. 查看评估结果，体验完整流程

### 场景2：查看行业趋势

1. 在首页点击「趋势洞察」或点击热门岗位
2. 切换「需求趋势」查看岗位需求变化
3. 切换「技能词云」查看热门技能
4. 切换「岗位列表」搜索具体岗位

### 场景3：基于历史评估继续规划

1. 进入「我的」→「评估历史」
2. 点击之前的评估记录
3. 选择之前看过的岗位
4. 点击「查看职业规划」或「查看培养方案」

### 场景4：对比不同岗位

1. 在评估结果页，点击不同岗位tab
2. 查看每个岗位的匹配度和缺口
3. 选择最感兴趣的方向深入了解

## 6. 接口测试

如果需要调试后端API，可以使用以下方法：

### 6.1 使用浏览器测试GET请求

```
http://localhost:3000/api/data/overview
http://localhost:3000/api/data/trend
http://localhost:3000/api/data/skill-cloud
```

### 6.2 使用 curl 测试 POST 请求

```bash
# 测试登录接口
curl -X POST http://localhost:3000/api/auth/login ^
     -H "Content-Type: application/json" ^
     -d "{\"code\":\"test_code\"}"
```

### 6.3 使用 Postman/Apifox

1. 创建新请求
2. 设置请求方法：GET 或 POST
3. URL：`http://localhost:3000/api/xxx`
4. 如果是 POST，添加 Header：`Content-Type: application/json`
5. 如果需要鉴权，添加 Header：`Authorization: Bearer <token>`

### 6.4 常用测试接口

## 7. 数据管理

### 7.1 查看数据库数据

使用 DB Browser for SQLite 或命令行：

```bash
# 使用命令行打开数据库
sqlite3 backend/data/ai_career.db

# 查看用户数量
SELECT COUNT(*) FROM users;

# 查看岗位数量
SELECT COUNT(*) FROM job_data;

# 查看评估记录
SELECT * FROM assessments ORDER BY created_at DESC LIMIT 5;

# 退出
.quit
```

### 7.2 重新初始化模拟数据

如果需要清空并重新生成数据：

```bash
cd crawler

# 删除现有数据库文件（会重新创建）
rm backend/data/ai_career.db

# 重新生成模拟数据
python seed_data.py
```

### 7.3 手动添加测试用户

```sql
-- 手动插入一个测试用户
INSERT INTO users (openid, nickname) VALUES ('test_openid_123', '测试用户');

-- 插入测试评估数据
INSERT INTO assessments (user_id, answers, personality_result, overall_result, score)
VALUES (
  1,
  '{"major":"工商管理","personality":{"type":"组织型","answers":[1,2,3,4,1]}}',
  '{"type":"组织型","description":"善于组织协调"}',
  '{"summary":"适合管理类岗位"}',
  85
);
```

### 7.4 数据备份

```bash
# 备份数据库文件
cp backend/data/ai_career.db backup/ai_career_$(date +%Y%m%d).db

# 恢复数据库
cp backup/ai_career_20260423.db backend/data/ai_career.db
```

## 8. 注意事项

### 8.1 关于大模型调用

- AI评估功能需要有效的 API Key
- 如果 API Key 额度用完，评估功能将无法使用
- 通义千问有免费额度，智谱GLM也有免费额度，DeepSeek也有免费额度

### 8.2 关于数据安全

- `.env` 文件包含敏感信息，不要上传到公开仓库
- 生产环境务必使用 HTTPS
- 定期备份数据库文件

### 8.3 关于性能

- SQLite 适合读多写少的场景，单文件数据库
- 高并发场景建议使用 Redis 缓存
- 大规模数据建议迁移到 PostgreSQL 或 MySQL
