## 1. 接口规范说明

### 1.1 通用规范

| 项目 | 规范 |
|------|------|
| 协议 | HTTP/HTTPS |
| 请求格式 | JSON |
| 响应格式 | JSON |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601（`2026-04-23T10:30:00Z`） |
| 鉴权方式 | Bearer Token（JWT） |
| 分页参数 | `page`（页码，从1开始）、`pageSize`（每页条数，默认10） |

### 1.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | number | 状态码，200表示成功 |
| message | string | 响应消息 |
| data | object/array/null | 响应数据 |

### 1.3 鉴权说明

需要鉴权的接口，请求头需携带：
```
Authorization: Bearer <token>
```

## 2. 认证模块接口

### 2.1 微信登录

**POST** `/api/auth/login`

微信小程序登录，用code换取token。

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |

```json
{

}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

      "userId": 1,
      "nickname": "微信用户",
      "avatar": "https://...",
      "isNewUser": true
    }
  }
}
```

## 3. 评估模块接口

### 3.1 提交评估信息

**POST** `/api/assessment/submit`

**需要鉴权**

提交用户信息，触发AI评估。

**请求参数：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| major | string | 是 | 所学专业 |
| personality | object | 是 | 性格特质 |
| personality.type | string | 是 | 性格类型（如"分析型"/"社交型"/"创造型"/"组织型"） |
| personality.answers | array | 是 | 问卷答案，5道题的选项 [1,2,1,3,2] |
| workPreference | object | 是 | 工作偏好 |
| workPreference.workMode | string | 是 | 工作模式偏好（"remote"/"office"/"hybrid"） |
| workPreference.companyType | string | 是 | 公司类型偏好（"big"/"startup"/"state"/"foreign"） |
| workPreference.workStyle | string | 是 | 工作风格偏好（"stable"/"challenge"/"balanced"） |
| resume | object | 否 | 简历要点 |
| resume.internships | string | 否 | 实习经历 |
| resume.projects | string | 否 | 项目经验 |
| resume.skills | string | 否 | 技能特长 |

**请求示例：**
```json
{
  "major": "工商管理",
  "personality": {
    "type": "组织型",
    "answers": [1, 2, 1, 3, 2]
  },
  "workPreference": {
    "workMode": "hybrid",
    "companyType": "big",
    "workStyle": "challenge"
  },
  "resume": {
    "internships": "某互联网公司HR实习生3个月",
    "projects": "校园招聘管理系统（负责需求分析）",
    "skills": "Excel、PPT、基础Python"
  }
}
```

**响应示例：**
```json
{

  "message": "评估完成",
  "data": {
    "assessmentId": 1,
    "resilienceScore": {

        "learningAbility": 85,
        "innovation": 72,
        "collaboration": 86
      }
    },
    "matchedJobs": [
      {
        "jobTitle": "HRBP",
        "matchScore": 89,
        "matchReasons": [
          "组织型性格与HRBP岗位高度契合",
          "HR实习经历为HRBP工作提供实践基础",
          "沟通协作能力突出，适合HRBP的跨部门协调角色"
        ],
        "gapSkills": ["数据分析", "AI招聘工具使用"],
        "gapAbilities": ["数据驱动决策", "组织诊断"]
      },
      {
        "jobTitle": "运营经理",
```

## 7. 错误码定义

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或token过期 |
| 403 | 无权限访问 |
| 404 | 资源不存在 |
| 429 | 请求频率过高 |
| 500 | 服务器内部错误 |
| 1001 | 微信登录失败 |
| 1002 | 用户不存在 |
| 2001 | 评估信息不完整 |
| 2002 | 大模型调用失败 |
| 2003 | 评估记录不存在 |
| 3001 | 规划生成失败 |
| 4001 | 数据查询失败 |

**错误响应示例：**
```json
{
  "code": 2001,
  "message": "评估信息不完整：专业信息不能为空",
  "data": null
}
```
