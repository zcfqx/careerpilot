## 1. 项目背景与目标

### 1.1 项目背景

当前就业市场面临AI技术冲击，管理类岗位的技能需求正在发生深刻变化。应届毕业生和职场新人缺乏对市场趋势的准确认知，难以判断自身能力与岗位需求的匹配程度，更缺乏个性化的职业发展指导。

### 1.2 项目目标

构建一个基于招聘大数据和AI大模型的微信小程序，为用户提供：
1. **数据驱动的岗位趋势洞察**——基于真实招聘数据的岗位需求分析
2. **AI驱动的韧性评估与岗位适配**——结合用户画像与岗位能力模型的智能匹配
3. **个性化职业发展路径规划**——短中长期目标与里程碑设计
4. **能力缺口培养方案**——针对缺口能力的具体学习建议

### 1.3 核心价值主张

**"让AI帮你看见未来职场，让数据指引你的成长路径"**

## 2. 用户角色定义

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 求职学生 | 应届毕业生/在校生，寻找管理类岗位 | 了解岗位需求、评估自身适配度、获取职业规划 |

## 3. 功能需求详细设计

### 3.1 模块一：招聘大数据趋势洞察

#### 3.1.1 数据采集（爬虫子系统）

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| C-01 | 岗位信息采集 | 采集前程无忧、智联招聘、BOSS直聘的管理类岗位信息 | 高 |
| C-02 | 关键词搜索 | 支持按关键词（行政经理、HRBP、运营经理等）+经验要求（应届生/不限经验）搜索 | 高 |
| C-03 | 数据清洗 | 去重、格式统一、异常值处理 | 高 |
| C-04 | 定时采集 | 支持定时自动采集（如每周一次） | 中 |
| C-05 | 数据导出 | 将清洗后的数据导出为JSON/Excel格式 | 中 |

#### 3.1.2 数据分析

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| A-01 | 岗位需求变化分析 | 统计各岗位的招聘数量变化趋势 | 高 |
| A-02 | 技能频率分析 | 统计AI/数据相关技能在岗位描述中的出现频率 | 高 |
| A-03 | 能力要求趋势分析 | 提取高频能力关键词（判断决策、AI工具使用、跨域整合等） | 高 |
| A-04 | 薪资分布分析 | 统计各岗位薪资区间分布 | 中 |
| A-05 | 地域分布分析 | 统计岗位在不同城市的分布情况 | 低 |

#### 3.1.3 趋势图表展示

| 编号 | 功能 | 描述 | 优先级 |
|------|------|------|--------|
| V-01 | 需求趋势图 | 展示岗位需求增长/下降曲线图 | 高 |
| V-02 | 技能词云 | 展示岗位技能关键词词云 | 高 |
| V-03 | 能力雷达图 | 展示岗位能力要求雷达图 | 高 |
| V-04 | 薪资分布图 | 展示薪资区间分布柱状图 | 中 |

## 5. 页面流程与交互设计

### 5.1 页面清单

| 页面 | 路径 | 说明 |
|------|------|------|
| 启动页 | /pages/splash | 品牌展示，自动跳转 |
| 首页 | /pages/index | 趋势洞察总览、快捷入口 |
| 趋势详情 | /pages/trend | 岗位趋势图表详情 |
| 评估入口 | /pages/assessment | 用户信息填写表单 |
| 评估结果 | /pages/result | AI评估结果展示 |
| 岗位详情 | /pages/job-detail | 单个岗位的详细匹配信息 |
| 职业规划 | /pages/career | 个性化职业发展路径 |
| 培养方案 | /pages/training | 能力培养推荐方案 |
| 个人中心 | /pages/profile | 个人信息与历史记录 |

### 5.2 核心用户流程

```
用户打开小程序
    ↓
启动页（品牌展示2秒）
    ↓
首页（趋势洞察总览 + 快捷入口）
    ├──→ 趋势详情页（查看岗位趋势图表）
    │       ↓
    │    返回首页
    │
    └──→ 点击「开始评估」
            ↓
        评估入口页（填写专业/性格/偏好/简历）
            ↓
        点击「提交评估」→ Loading动画
            ↓
        评估结果页（韧性得分 + 适配岗位列表）
            ├──→ 点击某个岗位 → 岗位详情页（匹配度+缺口能力）
            │       ↓
            │   点击「查看规划」→ 职业规划页（短中长期目标）
            │       ↓
            │   点击「培养方案」→ 培养方案页（课程/资源推荐）
            │
            └──→ 返回结果页
```

### 5.3 关键交互说明

#### 评估表单交互
1. 专业信息：提供常见专业列表供选择，支持搜索
2. 性格特质：5道选择题，每题2-3个选项，滑动选择
3. 工作偏好：标签式多选（点击选中/取消）
4. 简历要点：文本输入框，提供示例提示文字

#### 评估结果交互
1. 韧性得分：环形进度条动画展示
2. 适配岗位：卡片列表，每张卡片显示岗位名称、匹配度百分比、核心匹配理由
3. 点击卡片展开详情

#### 趋势图表交互
1. 折线图：支持点击数据点查看具体数值
2. 词云：支持点击关键词查看相关岗位
3. 雷达图：支持切换不同岗位的能力模型对比

## 6. 数据流程设计

### 6.1 整体数据流

```
[招聘网站] 
    ↓ (爬虫采集)
[原始数据] → [数据清洗] → [结构化数据]
                                ↓
                        [数据分析引擎]
                                ↓
                    [岗位能力基准库] ← 存入MySQL
                                ↓
[用户输入] → [AI评估引擎] → [大模型API]
                ↓                   ↓
            [用户画像]         [能力匹配分析]
                ↓                   ↓
            [评估结果] ← ← ← ← ← ←
                ↓
        [适配岗位列表 + 匹配度]
                ↓
        [职业发展规划] ← [大模型API]
                ↓
        [能力培养方案] ← [大模型API]
```

### 6.2 大模型调用流程

```
1. 用户提交评估信息
2. 后端组装Prompt：
   - 系统提示词（角色设定+任务描述）
   - 岗位能力基准数据（从数据库查询）
   - 用户画像数据（专业/性格/偏好/简历）
3. 调用大模型API
4. 解析大模型返回的JSON结果
5. 存入数据库 + 返回前端
```
